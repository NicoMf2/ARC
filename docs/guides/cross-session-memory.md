# Cross-Session Memory Guide

ArcKit includes a cross-session memory system that captures key decisions, vendor insights, research findings, and session summaries across Claude Code sessions. This allows Claude to recall what happened in previous sessions and build on prior work rather than starting from scratch.

> **Bundled Integration**: The memory MCP server is included in the ArcKit plugin and requires no additional configuration beyond having Node.js installed.

---

## How It Works

The memory system has two parts:

1. **Session Learner** (`hooks/session-learner.mjs`) — fires when a session ends, analyses recent git commits, and writes a JSON manifest summarising what happened
2. **Session Start** (`hooks/arckit-session.mjs`) — fires when a new session begins, reads pending manifests, and surfaces them as context instructing Claude to store them via the MCP memory server

This two-stage approach is necessary because **MCP tools are not available inside hooks** (hooks are Node.js scripts, not Claude tool calls). The session learner writes manifests to disk; the session start hook reads them and presents them to Claude for processing.

### Data Flow

```
Session N ends
  └── session-learner.mjs analyses git commits
       └── writes .arckit/memory/pending-{session_id}.json

Session N+1 starts
  └── arckit-session.mjs reads pending manifests
       └── surfaces context: "Previous session created REQ v1.0..."
            └── Claude calls mcp__memory__create_entities
                 └── entities stored in .arckit/memory.jsonl
                      └── Claude calls mcp__memory__search_nodes
                           └── retrieves relevant prior context
```

## Entity Types

The memory system uses seven entity types designed for architecture governance workflows:

| Entity Type | What It Captures | Example Observations |
|---|---|---|
| `ProjectDecision` | Key ADR outcomes that should persist | `["chose-kafka", "rejected-rabbitmq", "3yr-tco-£450k"]` |
| `VendorInsight` | Vendor evaluation findings | `["hidden-egress-costs", "no-uk-dc", "poor-gov-cloud-support"]` |
| `ReviewOutcome` | HLD/DLD review results | `["approved-with-conditions", "missing-dr-strategy"]` |
| `ResearchFinding` | Market observations | `["g-cloud-no-federated-identity", "all-vendors-use-oauth2"]` |
| `RecurringRequirement` | Patterns across projects | `["data-sovereignty-nfr-in-4-of-5-projects"]` |
| `LessonLearned` | Operational discoveries | `["sobc-exceeds-32k-tokens", "research-needs-req-doc-first"]` |
| `SessionSummary` | What happened in a session | `["created-req-v1", "3-adrs", "hld-review-pending"]` |

### Entity Naming Conventions

- **SessionSummary**: `session-YYYYMMDD-HHMM` (auto-generated by session learner)
- **ProjectDecision**: `decision-{project-id}-{short-description}` (e.g., `decision-001-kafka-over-rabbitmq`)
- **VendorInsight**: `vendor-{vendor-name}` (e.g., `vendor-aws`, `vendor-azure`)
- **ReviewOutcome**: `review-{artifact-type}-{date}` (e.g., `review-hld-20260220`)
- **ResearchFinding**: `research-{topic}` (e.g., `research-g-cloud-identity`)
- **RecurringRequirement**: `recurring-{pattern}` (e.g., `recurring-data-sovereignty`)
- **LessonLearned**: `lesson-{short-description}` (e.g., `lesson-sobc-token-limit`)

## Design Principles

### Memory Owns Learnings, Not Artifact Copies

The memory system stores **observations about decisions and outcomes**, not copies of artifacts. If you need the full content of an ADR, read the file. Memory captures what matters for cross-session continuity:

- "We chose Kafka because of the throughput requirement" (memory)
- Not: the entire ADR document text (that lives in the file system)

### Keep Entity Count Low

The `read_graph` MCP tool returns **all entities with no filtering**. The practical cap is approximately 50 entities before context window impact becomes noticeable. To stay within this limit:

- Use `SessionSummary` entities for automatic session tracking (prune after 30 days)
- Use specific entity types (`ProjectDecision`, `VendorInsight`, etc.) for persistent learnings
- Periodically consolidate related observations into fewer entities

### Project-Local Storage

The memory file lives at `.arckit/memory.jsonl` within the project directory, not globally. This means:

- Each project has its own memory — no cross-contamination
- Memory travels with the project (committed or gitignored, your choice)
- Different team members can share project memory via version control

## Session Classification

The session learner automatically classifies sessions based on which artifact types were created or modified:

| Session Type | Triggered By | Purpose |
|---|---|---|
| `governance` | Principles, Secure by Design, TCoP, DPIA, AI Playbook, JSP 936 | Security and compliance work |
| `research` | Research, AWS/Azure/GCP Research, G-Cloud Search | Market analysis and technology evaluation |
| `procurement` | Statement of Work, Evaluation Criteria, DOS, G-Cloud Clarifications | Procurement document preparation |
| `review` | HLD Review, DLD Review, Service Assessment | Design review and assessment |
| `general` | All other artifact types | General architecture work |

## When to Write to Memory

Claude should write to MCP memory **inline** during sessions, not just at session end. Use these triggers:

| Trigger | Entity Type | Example |
|---------|-------------|---------|
| ADR outcome decided | `ProjectDecision` | Chose event-driven over request-reply |
| Vendor evaluation finding | `VendorInsight` | Azure has no UK sovereign cloud option |
| Design review completed | `ReviewOutcome` | HLD approved with 3 conditions |
| Research finding | `ResearchFinding` | All G-Cloud vendors require custom IdP |
| Pattern observed 2+ times | `RecurringRequirement` | Data sovereignty NFR in every project |
| Bug or workaround found | `LessonLearned` | SOBC template exceeds 32k token limit |

### How to Write

1. **Search first** — `mcp__memory__search_nodes` with the key concept
2. **If entity exists** — `mcp__memory__add_observations` to append
3. **If new** — `mcp__memory__create_entities` with a keyword-searchable name

### Do Not Write

- Copies of artifact content (read the file instead)
- Trivial observations (typos, formatting fixes)
- Transient session context that will not matter tomorrow
- Duplicates of what was already captured this session

## Size Management

| Rule | Value | Rationale |
|------|-------|-----------|
| Entity cap | ~50 per project | `read_graph` returns everything; context impact above 50 |
| Max observations per entity | 15 | Condense or split if approaching |
| SessionSummary retention | 20 most recent | Auto-pruned by `memory-prune.mjs` |
| Persistent types | Never auto-pruned | `ProjectDecision`, `VendorInsight`, `ReviewOutcome`, `ResearchFinding`, `RecurringRequirement`, `LessonLearned` |
| Archive | `archive.md` | Pruned entities preserved for historical reference |

### Pruning

Run periodically to keep memory lean:

```bash
# Preview what would be pruned
node arckit-plugin/scripts/memory-prune.mjs --dry-run

# Prune keeping last 10 session summaries
node arckit-plugin/scripts/memory-prune.mjs --keep 10

# Default: keep last 20
node arckit-plugin/scripts/memory-prune.mjs
```

### Searching

Search across both active memory and the markdown archive:

```bash
# Search for a term
node arckit-plugin/scripts/memory-search.mjs "kafka"

# Filter by entity type
node arckit-plugin/scripts/memory-search.mjs "kafka" --type ProjectDecision

# Show statistics
node arckit-plugin/scripts/memory-search.mjs --stats
```

## Troubleshooting

| Symptom | Cause | Fix |
|---------|-------|-----|
| No memory entities after session | Session had no git commits | Memory only captures sessions with commits |
| Memory not surfaced at session start | No pending manifests | Check `.arckit/memory/` for `pending-*.json` files |
| `mcp__memory__*` tools not found | MCP server not configured | Verify `memory` entry in `.mcp.json` |
| `mcp__memory__*` tools need permission | Auto-allow not matching | Verify `mcp__memory__` prefix in `allow-mcp-tools.mjs` |
| Memory file too large | Too many entities | Run `memory-prune.mjs` to archive old sessions |
| Search returns nothing | Entity was pruned | Search includes archive — check `--stats` for counts |

## Files

| File | Purpose |
|------|---------|
| `arckit-plugin/.mcp.json` | MCP server configuration (includes memory) |
| `arckit-plugin/hooks/session-learner.mjs` | Stop hook: captures session data as manifests |
| `arckit-plugin/hooks/arckit-session.mjs` | Start hook: surfaces pending manifests |
| `arckit-plugin/hooks/allow-mcp-tools.mjs` | Auto-allows `mcp__memory__*` tools |
| `arckit-plugin/scripts/memory-search.mjs` | Search across active memory and archive |
| `arckit-plugin/scripts/memory-prune.mjs` | Prune ephemeral entities with archival |
| `.arckit/memory.jsonl` | Active memory store (project-local) |
| `.arckit/memory/pending-*.json` | Session manifests awaiting processing |
| `.arckit/memory/archive.md` | Archived pruned entities |
